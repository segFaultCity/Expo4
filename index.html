API Test/Practice
1.Google Maps API
You must have location services enabled for Geolocation to work. If you have this turned off, this site will not display anything. Zoom in to see your exact location once you allow location service!

What I added/Implemented:
  • For the first step, I added the basic google maps API via a link in the html file.
  • To grant access to this API, I had to create an API key through the developers console.
  • To add more functionality, I added the Geolocation service (pinpoints your location and displays it on their map) that Google offers with the Maps API. Luckily, there was a lot of documentation to help!
  • Lastly, for this segment, I added Google Maps Markers to show your current location in accordance with the University of Missouri. This was unbelievably easy to add though after reading a short document on it.
2.Street View Platform
This street view was implemented to start from your current location, so location services must be allowed! Check out your surroundings...Pretty familiar huh? (This view may be a street or a block off based on your current location. Check the streets on the Map above and compare it to the one labeled at the top left for the street view and make sure you're close to your current location!)

Implementing this platform:
  • The Street view was really simple to add and took far fewer lines of code to incorporate after adding the Google Map.
  • After already configuing a users current location, it was simple to make the street view start near that exact position.
